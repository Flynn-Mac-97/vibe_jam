# Scope 看板 · 必做 / 应做 / 可做 / 不做

动态文档。廖琦在每个 gate（每次试玩后、Lock 时）更新。贴墙或共享屏。每个人在拿任务前都要看一眼。

## 看板

```
┌───────────────────────┬───────────────────────┐
│ MUST 必做             │ SHOULD 应做           │
│ (缺了 demo 就废)      │ (有余力就做)          │
│                       │                       │
│ - …                   │ - …                   │
│ - …                   │ - …                   │
│                       │                       │
├───────────────────────┼───────────────────────┤
│ COULD 可做            │ WON'T 不做            │
│ (有更多余力再说)      │ (明确排除)            │
│                       │                       │
│ - …                   │ - …                   │
│ - …                   │ - …                   │
└───────────────────────┴───────────────────────┘
```

## 定义

- **MUST 必做**：Lock 时若缺一项，demo 就废。核心循环、失败态、分数、重开、1 个主视觉、1 个主音效。
- **SHOULD 应做**：把 MUST 从"能玩"变成"像样 demo"的二级功能 —— 反馈增益、提示文字、关键反馈音、设置。MUST 做完才做。
- **COULD 可做**：SHOULD 做完后才考虑的增强。8 小时 jam 里很少会到这层。
- **WON'T 不做**：明确不在范围内。写下它们和写 MUST 同样重要 —— 防止静默 scope 蔓延。

## 规则

- 把一项从 SHOULD 挪到 MUST 要廖琦批准，并做交换（把一条 MUST 降到 SHOULD）。
- 加新项要交换或删除。不允许静默添加。
- WON'T 不能回来。有人提"要不要加..." —— 指着 WON'T 栏。
- 每个 gate 重审看板。如果试玩显示某项没承载感，可以降级（MUST → SHOULD → COULD）。

## 示例（Playbook A · 主题 "循环"）

```
MUST
 - 玩家用方向键 / WASD 移动（TopDown）
 - 星星从上方生成、下落、拾取加分
 - 威胁从上方生成、下落、碰到 GameOver
 - GameOver 页有分数、最高分、重开
 - 拾取 SFX + 碰撞 / 失败 SFX
 - BGM 循环（trixie）

SHOULD
 - 碰撞时屏幕震动
 - "第几轮" 的 UI 计数（贴主题）
 - 每轮难度曲线更陡
 - 主视觉星星（比占位更亮）

COULD
 - 金色星星 5 倍分、存在时间短
 - 擦身而过的 slow-motion
 - 标题淡入

WON'T
 - 多难度模式
 - 强化道具
 - 第二个场景
 - 暂停功能（P 键退出足够）
 - 玩家血条
```
