# MVP 定义模板

收敛结束（第 15 分钟）立即填完。一页，不超。这是周日全天的契约。

```
Demo 名（工作名）：       [可在 Lock 前修改]

主题（公布原文）：        [主题字符串]

选中的 Playbook：         [A / B / C]

一句话定义：
  在 [场景]，玩家 [核心动作] [目标] 来 [目的]，同时 [压力]。

核心体验（1 行）：
  [玩家"感受到什么"，不是"做什么" —— 比如 "多待一秒的张力"]

一行 hook（用于 demo 页 / 直播标题）：
  [≤ 80 字符，标题风格]

胜利条件：               [或 "无明确胜利；分数驱动"]

失败条件：               [一击 / 漏 3 次 / 时间到 / ...]

单局时长目标：           [典型 30-90 秒]

包含的系统：
  - [最多 5 条，每条是一个 kit 模板或明确的游戏动词]
  -

明确不做的系统：
  - [最多 5 条，刻意不做的]
  -

主视觉（1 项）：         [最能卖 demo 的那张美术 —— 比如 "得分区时被放大的月亮"]

主音效（1 项）：         [最能卖 demo 的那个瞬间音效或 BGM 段落]

3 必做（收敛结果）：
  1.
  2.
  3.

3 不做（收敛结果）：
  1.
  2.
  3.

首席设计：                廖琦
场景 owner：              Flynn
音频 owner：              trixie
美术 owner：              mengping
程序对：                  Flynn + David
```

## 规则

- 每个字段都要填。空白就是 scope 风险。
- 不适用就明确写 `n/a`。不要静默省略。
- 把这页贴在所有人都能看到的位置。
- 第 15 分钟后廖琦修改任何字段，必须口头宣布，所有人同步更新在看的副本。
