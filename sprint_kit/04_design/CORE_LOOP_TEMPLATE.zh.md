# 核心循环模板

写三个时间尺度的玩家体验。如果三层都无法写出具体内容 —— 循环还没成立。

```
10 秒层（第一印象）：
  - 玩家感知到什么：
  - 玩家第一件会尝试的事：
  - 第一次正反馈（奖励这次尝试的信号）：

60 秒层（进入技巧区）：
  - 引入的新元素：
  - 第一次有意义的决策：
  - 第一次非 trivial 的失败可能：

3 分钟层（掌握区）：
  - 升压（什么东西在持续变难）：
  - 玩家在追打什么：
  - "差一点"的瞬间感觉是什么：
```

## 验证规则

- 每一层都要至少一个具体名词 + 一个具体动词。"玩家感到紧张"不具体。"陨石擦过玩家 2 个单位、分数跳到 45"才具体。
- 10 秒层的正反馈必须在首次游玩的前 10 秒**可见 + 可听**。拖到 15 秒不算。
- 60 秒的决策必须是真 trade-off，不是对错答案。
- 3 分钟的升压必须来自 60 秒时就已经存在的东西 —— 不能在 3 分钟时引入新系统。

## 示例（Playbook A · 主题 "循环"）

```
10 秒层：
  - 感知：月亮下一个角色站在地上；星星开始从上方下落。
  - 第一反应：左右走（平台游戏直觉）。
  - 第一次正反馈：碰到星星，分数从 0 跳到 10，带一个明亮的拾取音效。

60 秒层：
  - 新元素：陨石和星星一起下落。
  - 决策：去右边抓漂移的星星（那里陨石也在逼近），还是在中间稳吃低价值的星星。
  - 失败：碰一下就 Game Over；测试者切身感受到后果。

3 分钟层：
  - 升压：生成速率按 difficultyCurve 陡增；屏幕变密；反应时间被压缩。
  - 追打：个人最高分 + "这轮循环我能活多久"。
  - 差一点的瞬间：陨石擦身而过半单位，拿到了星星，分数跳一下 —— 这种刺激推动下一局。
```

## 反模式（看到就重写）

- 10 秒层里写"玩家阅读教程" —— 不要教程。只允许直觉或一行提示。
- 60 秒的决策写成"玩家变得更熟练" —— 那不是决策，那是成长。
- 3 分钟层引入了强化道具系统 —— scope 爆炸。
