# 主题收敛 · 15 分钟协议

廖琦主持，所有人都能看到计时器。第 15 分钟结束还没决策 —— 廖琦直接拍板，不再重开。

## 输入

- 主办方公布的主题（1-2 个词）
- 三份 Playbook：`04_design/PLAYBOOK_A/B/C.en.md`（和 `.zh.md` 孪生版，仅廖琦看）
- 已有的夜空场景与资产

## 15 分钟怎么过

### 第 0-3 分钟 · 主题解读（3 分钟）

静默书写（全员）。每人写 3 条"一句话主题解读"。不讨论、不评论。

模板：
- 解读 1：…
- 解读 2：…
- 解读 3：…

第 3 分钟廖琦收集。不要全念出来 —— 挑最不一样的 5 条，重复的丢掉。

### 第 3-8 分钟 · 映射到 Playbook（5 分钟）

在共享视图里（白板 / 共享文档），廖琦把 5 条解读放到一个 3 列表：

| 解读 | 最匹配的 Playbook | 原因（1 行） |
|---|---|---|
| … | A / B / C | … |

约束：每条解读只分配给一个 Playbook。如果一条能套两个，选更可靠的（可靠性排序：A > C > B）。

第 8 分钟统计：哪个 Playbook 拿到的解读最多？就是它。

打平规则：平局时 A 胜。A 和 B 平 → A。A 和 C 平 → A。B 和 C 平 → B。（A 是最可靠的路径，默认选可靠。）

### 第 8-13 分钟 · 落定 3 必做 / 3 不做（5 分钟）

打开选中的 Playbook。它原本的"3 必做 / 3 不做"就是起点。

全队把两个列表**加上主题特定的内容**重写：

**必做**（刚好 3 条）：
1. …
2. …
3. …

**不做**（刚好 3 条）：
1. …
2. …
3. …

这 3+3 是接下来全天的契约。任何人想加一条 —— 就必须交换：拿掉一条旧的才能加新的。

### 第 13-15 分钟 · 一句话定义 + 开工（2 分钟）

廖琦在板上写一句：

```
在 [场景]，玩家 [核心动作] [目标] 来 [目的]，同时 [压力]。
```

示例（主题 "循环" + Playbook A）：
```
在循环的夜空中，玩家收集下落的星星追打最高分，同时上一轮的陨石在本轮下落得更快。
```

读出来一次。全员确认或提一次反对意见。廖琦决定。

第 15 分钟：Flynn 开始导入脚本。David 开始根据 playbook 搭建需要的脚本。mengping 挑一张现有资产作为主视觉。trixie 开始做 `BgmKey.Main` 的 BGM。

## 规则

- 第 15 分钟前没人开编辑器、没人写代码。先收敛。
- 这 15 分钟手机不上场。
- 如果主题感觉完全套不进 —— 选 Playbook A 找任何隐喻即可。出得了 demo 胜过聪明。
- 第 15 分钟后有人说"要不我们完全换一个方向" —— 答案是不行。

## 收敛演习（周六晚）

廖琦随机挑一个假主题（"共生"、"碎片"、"循环"、"光"、"小即是大"）。全队跑完 15 分钟。如果演习时第 15 分钟还没落定 —— 重跑演习。真的周日当天没有余量。
